Для исполнения скрипта нужно выполнить несколько действий:
1. Необходимо сменить источник загрузки. Для этого используйте vim и закоментируйте/раскоментируйте нужные строки в /etc/apt/sources.list.d/alt.list (закоментируйте все 3 строки) и в /etc/apt/sources.list.d/yandex.list (раскоментируйте 3 строки http://mirror...))
2. выполните apt-get update для обновления списка пакетов
3. установите wget используя команду: apt-get install -y wget
4. c помощью wget скачайте нужный файл скрипта из репозитория используя следующий формат: wget https://raw.githubusercontent.com/DimaDemoExam/<номер модуля>/refs/heads/main/<имя файла>.txt (Пример: wget https://raw.githubusercontent.com/DimaDemoExam/Module1/refs/heads/main/1_ISP.txt)
5. сконфигурируйте необходимые значения в скрипте используя vim
6. скопируйте текст скрипта из файла (используя консоль xterm.js) используя выделение с помощью мыши (Примечание! Делайте это очень аккуратно и внимательно, так как консоль глючит (она листает вверх и дублирует текст по 2 раза и т.д.) и можно скопировать очень много лишнего!)
7. вставьте текст в консоль машины (используя правую кнопку мыши, нажав "вставить" из контекстного меню)

Примечание! Используйте isp как место хранения всех скриптов, и просто копируйте их в консоли других машин оттуда (используя 2 консоли xterm.js) В некоторых местах скрипта можно поменять значения на необходимые. После завершения работы со скриптами очистите оставшиеся файлы и историю команд на isp (инструкция в 9_Uninstall.txt). Чтобы не забыть порядок и названия файлов скриптов, просто первым делом скачайте 0_Instruction.txt на isp, и заглядывайте туда для подсказки (для каждого модуля 0_Instruction.txt свой). Я мог где-то что-то сделать неправильно, мог затупить, может что-то сломаться, могут заблокировать github, могут заблокировать консоль xterm.js на экзамене, может что-то ещё пойти не так во время экзамена и т.д., так что лучше всего пользуйтесь своей смекалкой в такие моменты.
